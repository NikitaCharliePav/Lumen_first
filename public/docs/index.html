Introduction

Start (and never finish) side projects with this API.

This documentation will provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

This API is not authenticated.

Orders.

Order

Создание заказа.

Возвращает объект класса заказ
При ошибки валидации возвращает статус 422. Если передан неизвестный продукт возвращает статус 404
Example request:
curl --request POST \
    "/api/v1/order/create" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"user_id\": \"384b0e3a-ce7f-4caf-8512-97cf1dcd0a11\",
    \"products\": [
        \"est\"
    ]
}"

const url = new URL(
    "/api/v1/order/create"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "user_id": "384b0e3a-ce7f-4caf-8512-97cf1dcd0a11",
    "products": [
        "est"
    ]
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());
$client = new \GuzzleHttp\Client();
$response = $client->post(
    '/api/v1/order/create',
    [
        'headers' => [
            'Content-Type' => 'application/json',
            'Accept' => 'application/json',
        ],
        'json' => [
            'user_id' => '384b0e3a-ce7f-4caf-8512-97cf1dcd0a11',
            'products' => [
                'est',
            ],
        ],
    ]
);
$body = $response->getBody();
print_r(json_decode((string) $body));
